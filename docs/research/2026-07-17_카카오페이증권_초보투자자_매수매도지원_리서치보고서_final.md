## 3. 문제 재정의

**양측 재정의가 실질 동일합니다** (C-보고서 3장 = X-CLM-0036, H — 독립 수렴):

> **가격 변동과 손실 때문에 판단이 흔들리는 경험 부족 투자자가 — ① 보유 종목 급락·손실 국면("지금 팔아야 하나"), ② 첫 매수 주문 순간, ③ 매도·매도 후 처리 국면에서 — [왜 이런 일이 벌어졌는지의 근거], [무엇을 확인해야 하는지의 절차], [이 선택의 비용·위험·비가역성]을 이해하고, 자신의 원래 계획·최신 사실·주문 결과를 대조하여 **스스로 설명 가능한 결정**을 내리도록 돕는 문제.** 해법은 결론 대행이 아니라 출처 있는 판단 재료의 구조화와 선택권의 반환이다.

### 8.3 아키텍처 (병합 — 폴백 3계층 × 계산·생성 분리)

**폴백 3계층**: ① 실제 수집 성공 후 해시·기준시각을 기록한 pykrx/KRX 스냅샷·DART 캐시 ② 로컬 모의체결 엔진 ③ 오프라인 JSON fixture·LLM 응답 캐시입니다. KIS 모의투자는 키·계좌·호출제한을 실제 확인한 경우에만 선택합니다.

**계산·생성 분리 원칙**(Codex, X-CLM-0030 채택): 숫자(체결·비용·세금·정산·잔여비중)는 **결정론적 계산기**가 담당하고, LLM은 근거가 붙은 사실의 쉬운 설명·질문 정리만 수행하며 정책 가드레일 뒤에 둡니다. 인용 가능한 원문이 없거나 데이터가 오래되면 답 대신 `확인 불가`·`15분 지연`·`장 마감 기준` 상태를 반환합니다.

**AI 응답 계약**(Codex §10.3 채택): 모든 AI 출력은 `facts(+source_id·as_of) / interpretations / unknowns / user_inputs / calculation_id / policy_result / next_questions` 구조를 갖추고, source_id 없는 외부 사실·미래 수익률 숫자·매수/매도 결론·사용자 입력과 다른 수량 제안은 **렌더링 전에 차단**합니다. 보안: 공시·뉴스·커뮤니티 텍스트는 명령이 아니라 데이터로 격리(프롬프트 인젝션 대비), 주문 자격증명은 모델 컨텍스트·로그에서 제거, 모델 실패 시에도 원문 카드·계산기는 동작.

**추가 운영 요건**: OHLCV·DART만으로 가격 하락의 '원인'을 확정할 수 없습니다. 뉴스·이벤트 원천·이용권·프로브넌스가 없으면 기능명을 '관련 사실 브리핑'으로 한정합니다(F-CLM-0018 취지). 비용·세금·결제일 계산기는 시장(KRX/NXT/해외)·상품·계좌·거래소·효력일 버전 규칙을 사용합니다. 현재 NXT·SOR 체계는 [최선집행기준](https://www.kakaopaysec.com/guide/bestExecutionStandard/dynamicPage.do)과 [수수료 안내](https://www.kakaopaysec.com/guide/feeGuide/dynamicPage.do)를 따라 예상 체결시장·가격·거래시간·비용에 반영합니다(F-CLM-0008, H).

### 9.2 핵심 데모 통합 정의 — **판단 여권: 고불안 순간 판단 전 브리핑**

제품명 '판단 여권(Decision Passport)'(Codex) + 데모 시나리오 '고불안 순간 판단 전 브리핑'(Claude). 시나리오: 보유 종목 -8% 급락 + 악재 뉴스.

1. **상황 선택·계획 회상**(Codex 추가): 처음 매수/추가 매수/손실 중/수익 중/매도 고민 중 선택 → 목표 기간·최대 감수 손실·재검토 조건 확인. 계획이 없으면 AI가 결론 대신 **질문 초안**을 생성.
2. **관련 사실 브리핑**: 출처·기준시각을 붙이고 `확인된 사실 / 해석 / 알 수 없음`을 분리합니다. 호재·악재로 판단되는 자료를 대칭 노출하되, 뉴스·이벤트 근거가 없으면 '원인'을 출력하지 않습니다.
3. **판단 체크리스트 + 이해 확인**: 확인 절차 5개 + "시장가 실제 체결가는 다를 수 있다" 등 핵심 조건을 짧은 질문으로 확인(Codex).
4. **대칭 시나리오 + 주문 영향 미리보기**: 유지/일부매도/전량매도를 어느 것도 기본 강조하지 않고 비교합니다. 시장·상품·계좌·거래소별 수수료·세금·실현손익·잔여비중·예상 결제일·예상 체결시장(KRX/NXT)을 표시합니다.
5. **검토 의향 선택**: `유지 검토` / `일부매도 검토` / `전량매도 검토` / `나중에 재검토`를 선택하고 이유를 기록합니다. 이 버튼은 주문 CTA가 아니며 AI 대화 안에 실행 버튼을 두지 않습니다. 추가매수는 사용자가 먼저 의향을 밝힌 경우에만 위험과 함께 열어줍니다.
6. **실행 분리**: 별도 주문 화면에서 종목·수량·주문유형 재확인 → 주문 미리보기 → **모의 체결**(실주문 없음).
7. **사후 기록**(Codex 추가): 결정 이유·재검토일 저장 — 결과가 아니라 과정의 일관성을 회고.
8. **엔딩 신 — 장애 안전모드**(D-08): 장애 시 신규 주문 유도 문구를 숨기고 주문 상태·영향 범위·공식 공지만 표시하는 화면 전환 시연 — VOC 최상위 불만("결정적 순간의 배신")에 대한 응답이자 차별 포인트.

화면 상수 4종: 무엇을 아는가(출처·기준시각) / 무엇을 모르는가 / 무엇이 달라지는가(현금·비중·비용·결제일·체결시장) / 누가 결정하는가(사용자)입니다. 예시 문구의 기준시각은 실제 스냅샷 동결 성공 후 자동 입력하며, 미래 시각을 예정 사실처럼 쓰지 않습니다.
