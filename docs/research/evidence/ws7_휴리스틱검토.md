## 1. 검토 방법

1. **휴리스틱 프레임**: 계획서 3장 WS7이 지정한 초보 투자자 관점 휴리스틱 4종(불안 완화 / 이해 가능성 / 오주문 방지 / 근거 제시)을 WS2~WS5 근거로 조작화했습니다(2장).
2. **공인 기준 결합**: WS5가 도출한 해커톤 데모 필수 안전장치 5종(출처·기준시각 배지, 결론 비워두기, 단정 표현 가드, 주문 미리보기 정지선, 접근성 기본기 — `ws5_규제_소비자보호.md` 4.2절, C-CLM-0513)과 금융소비자보호 기준(금소법 §17·§19·§21 [C-SRC-0505](https://www.law.go.kr/법령/금융소비자%20보호에%20관한%20법률/제17조)·[C-SRC-0507](https://www.law.go.kr/법령/금융소비자%20보호에%20관한%20법률/제19조)·[C-SRC-0508](https://www.law.go.kr/법령/금융소비자%20보호에%20관한%20법률/제21조)), 접근성 기준(장차법 §21 [C-SRC-0512](https://www.law.go.kr/법령/장애인차별금지%20및%20권리구제%20등에%20관한%20법률/제21조))에 비추어 후보 기능을 점검했습니다.
3. **검토 대상**: WS4 패턴 분석(4.1·4.2절)과 WS5 규제 매트릭스에서 도출된 후보 기능 6종 + 배제 대상 3종.
4. **판정 표기**: ◎ 충족(설계상 명확) / ○ 조건부 충족(설계 조건 준수 시) / △ 부분·주의 / ✕ 위배·부적합. 모든 판정은 해석(I) 또는 제품 결정(D)입니다.

## 5. WS5 안전장치 5종·접근성 기준 충족 점검

채택 기능 6종(F1~F6)을 하나의 데모 흐름("급락 브리핑 → 체크리스트 → 시나리오/비용 미리보기 → 3버튼 선택 → 주문 미리보기 → 모의 체결")으로 묶었을 때의 점검표입니다.

| WS5 안전장치 (ws5 4.2절, C-CLM-0513) | 충족 방안 | 담당 기능 |
|---|---|---|
| 1. 출처·기준시각 배지 | 모든 수치·요약에 출처명+딥링크+기준시각("2026-07-17 15:30 KST 종가 기준") 상시 표기. 폐장일 데모 정직성(C-CLM-0601) | F1·F3·F4·F5 전체 |
| 2. 결론 비워두기 패턴 | 응답 템플릿에서 방향 결론 슬롯 제거, 마지막 화면은 항상 사용자의 [매수]/[매도]/[보류] 동일 위계 3버튼 | F2 |
| 3. 단정 표현 가드 | 출력 전 필터로 "오릅니다/안전합니다/확실" 차단 + 상승/하락 재료 병기 강제 — 투자권유성 표현률 0을 시연 지표로 | F1·F3 |
| 4. 주문 미리보기 정지선 | 요약 확인 화면까지만 진행, 제출은 모의 체결 대체, 잔고 대비 과대 주문 경고 | F5 |

(신규 주장 C-CLM-0702: 채택 기능 6종의 결합 흐름은 휴리스틱 4종과 WS5 안전장치 5종을 구조적으로 충족하도록 설계 가능하다, D, 확신도 M)

## 7. 종합 판정

- **채택**: F1(급등락 원인 브리핑) + F2(판단 체크리스트·3버튼) + F4(매도 전 비용·세금·출금 미리보기) + F5(주문 미리보기·모의 체결)를 핵심 축으로, F3(시나리오 계산기)·F6(용어 즉시풀이)을 보조·공통 기반으로 결합하는 것이 휴리스틱 4종·안전장치 5종·접근성 기준을 모두 통과하는 유일한 조합입니다(I/D, C-CLM-0702).
- **배제**: X1(개인화 추천)·X2(권유형 순위)·X3(실주문)은 규제 적색 또는 소비자보호 위배 소지로 총점 무관 배제 — WS8 기회 평가에도 동일 적용(C-CLM-0513 준수).
- 이 조합은 인터뷰에서 확인된 출제 의도("손해 보지 않았나"의 불안 + "어떻게 해야 할지 모르겠다"의 막막함, `2026-07-16_카카오페이증권_인터뷰영상분석.md` 2.3절)와 회사 공개 전략('왜지·어쩌지' 3단계 정보 모델, C-CLM-0102·0116)에 정합합니다.

## 8. 한계

1. **실사용자 미검증**: 초보 투자자 1~2명 과업 테스트(계획서 3장 WS7 선택 항목)를 수행하지 못했습니다. 모든 판정은 문헌·VOC·간접 벤치마크 기반의 전문가 검토이며, 본선 전·당일에라도 팀 외 1인의 신선한 눈 테스트를 권고합니다.
2. **간접 벤치마크 의존**: WS4가 실기기 미실행 간접 조사이므로, 비교 대상 앱의 실제 화면과 차이가 있을 수 있습니다(WS4 0장 한계 승계).
3. **체크리스트 항목 수·문구 등 세부 설계값**(예: 5개 이내)은 근거 기반 추정(I)이며 실측 검증이 없습니다.
4. **접근성은 원칙 수준 점검**: KWCAG 원문 미확인(WS5 한계 승계), 스크린리더 전면 검증 미수행.
5. 휴리스틱 검토는 평가자(단일 AI 리서처) 편향 가능성이 있으며, WS9에서 Codex 측 검토와 교차 대조가 필요합니다.
