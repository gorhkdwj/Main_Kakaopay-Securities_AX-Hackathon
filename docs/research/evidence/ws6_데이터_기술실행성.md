## 3. 토요일(7/18) 폴백 데이터 아키텍처 — 기본안

### 3.1 설계 원칙

1. **본선 당일 외부 API 실시간 의존 0건**을 목표로 합니다. 모든 시세·재무·환율 데이터는 7/17(금) 장 마감 후 로컬 스냅샷으로 고정합니다.
2. 모든 화면·응답에 **기준시각(예: "2026-07-17 15:30 KST 종가 기준")과 출처**를 표기합니다. 이는 폐장일 데모의 정직성 확보이자 계획서 4.4의 "오래된 데이터 미표시율" 지표 충족 수단입니다.
3. 주문은 실제 체결 없이 **로컬 모의 체결 엔진**(스냅샷 가격 기준 가상 체결 + 수수료·세금 계산)으로 처리합니다.

### 3.2 3계층 구조

```text
[계층 1] 사전 캐싱 스냅샷 (기본 경로, 7/17 금 저녁 확정)
  ├─ 시세: pykrx 일별 OHLCV·시총·PER 등 (7/17 종가) → data/snapshot_krx_20260717.parquet(csv)
  ├─ 공식 검증본: KRX 정보데이터시스템 종가 CSV 수동 다운로드 → data/krx_official_20260717.csv
  ├─ 미국·환율(필요 시): yfinance 일별 종가 + ECOS/야후 환율 → data/snapshot_us_20260717.csv
  ├─ 재무·공시: DART OpenAPI 주요 종목 20~30개 사전 조회 → data/dart_cache/*.json
  └─ 메타: 스냅샷 생성 시각·출처 URL·기준시각을 manifest.json에 기록

[계층 2] 모의 체결 폴백 (주문 데모 경로)
  ├─ 기본: 로컬 모의 체결 엔진 — 스냅샷 종가 ± 시나리오 변동으로 가상 체결,
  │        주문 미리보기 → 최종 확인 → 체결 내역 생성 (실거래 없음, 계획서 1장 경계 준수)
  └─ 선택: KIS 모의투자 API (7/17까지 키 발급·토큰 테스트 완료 시에만, 주말 미개장이므로
           "주문 접수 데모" 수준으로 한정하고 실패 시 즉시 로컬 엔진으로 전환)

[계층 3] 오프라인 최종 폴백 (네트워크·회장 장애 대비)
  ├─ 계층 1 스냅샷 전체를 노트북 로컬 + USB/클라우드 사본으로 이중화
  ├─ 대표 시나리오 3종(첫 매수 / -8% 손실 / +15% 매도)의 데이터를 JSON fixture로 별도 고정
  └─ LLM 호출 실패 대비: 시나리오별 대표 응답 캐시(데모 스크립트 상 필수 화면만이라도 재현)
```

### 3.3 갱신 시점 주의

- 공공데이터포털 주식시세 API는 D+1 13시 이후 갱신이므로 **7/17 종가는 토요일에 조회 불가**합니다 [C-SRC-0610](https://www.data.go.kr/data/15094808/openapi.do). 7/17 종가는 pykrx(장 마감 후 당일 수집 가능) 또는 KRX 정보데이터시스템 CSV로 확보해야 합니다.
- pykrx 수집은 7/17 **18시 이후**(일별 통계 확정 후) 실행을 권장하고, 미국 시장 스냅샷은 7/17(금) 미국 장 마감이 한국시간 **7/18(토) 새벽 5시**(서머타임 기준)이므로, 미국 데이터를 쓸 경우 7/18 아침 최종 1회 갱신하거나 7/17 목요일 종가(7/16 미국 장) 기준임을 명시해야 합니다.

## 4. 권고 구성: 1안 / 2안 / 폴백

- **1안 (권장, 국내주식)**: `pykrx 7/17 종가 스냅샷 + DART 공시·재무 캐시 + 로컬 모의 체결 엔진`. 키 리스크 0(발급 실패로 데모가 막힐 일 없음), 라이선스 리스크 낮음, 토요일 완전 동작. KRX 정보데이터시스템 CSV를 공식 검증본으로 병행 보관해 신뢰성을 보강합니다.
- **2안 (미국주식 시나리오 채택 시)**: `yfinance 일별 스냅샷 + ECOS 환율 + 로컬 모의 체결 엔진`. 초보자 페인포인트(환율·시장시간) 소재는 풍부하나 yfinance 비공식·개인용 제약과 데이터 검증 원천 부족이 한계 — 계획서 1장 "공식 데이터와 오프라인 폴백이 안정적인 시장 우선" 기준상 1안 후순위입니다.
- **폴백 (공통 필수)**: 시나리오 3종 JSON fixture + manifest 기반 오프라인 데모. 네트워크가 완전히 끊겨도 데모 전체가 로컬에서 완주 가능해야 합니다. KIS 모의투자는 "있으면 가점, 없어도 무방"인 선택 요소로만 취급합니다.
