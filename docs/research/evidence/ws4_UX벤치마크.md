## 0. 조사 방법과 한계 (필독)

> ⚠️ **간접 조사 한계 명시**: 본 벤치마크는 **모바일 앱을 직접 실행하지 못한 상태**에서 수행한 간접 벤치마크입니다. 근거는 ① 공식 이용 가이드·FAQ(토스·카카오페이 고객센터는 Playwright로 JS 렌더링 후 원문 확인), ② 토스증권 웹 버전(WTS) 비로그인 직접 관찰, ③ 공식 블로그·보도자료, ④ 언론 보도, ⑤ 블로그·커뮤니티 사용기입니다. 따라서 **탭 수·판단 지점 수는 공식 경로 안내 기반 추정치**이며, 실제 화면 단위 수치는 6장의 실기기 확인 체크리스트로 참가자가 별도 검증해야 합니다. 확인하지 못한 항목은 `미확인`으로 표기했으며 추정으로 채우지 않았습니다.
- 유튜브 워크스루 영상은 시간 제약상 개별 영상 정독(타임스탬프 기록)까지 진행하지 못했습니다. 실기기 확인이 어려울 경우의 차선책으로 체크리스트에 포함했습니다.
- 로보어드바이저(핀트·에임·Betterment)는 직접 매매 개념이 없어 과업 ①~④를 "해당 과업이 어떤 형태로 치환되는가" 관점으로 비교했습니다.
- 미국 서비스(Robinhood·Betterment·Magnifi)는 국내 미제공으로 웹 공식 문서 기반이며, 과업 ④(환율)는 해당 없음(자국 통화 거래)입니다.

## 4. 패턴 분석 — 초보자 관점

### 4.1 잘하는 패턴 (데모에 차용할 것)

1. **쉬운 언어 치환**: 매수/매도 → 구매/판매(토스·카카오페이 공통), 시장가 → '최적가'(카카오페이) [C-CLM-0401, 0402]. 단, 4.2-④의 부작용과 세트로 봐야 합니다.
2. **금액 기반 주문 기본값**: Robinhood 달러 입력 기본 [C-CLM-0407], 카카오페이 1,000원 소수점·원화 주문 [C-CLM-0403], 토스 총 주문 금액 입력 [C-CLM-0404]. "1만 원어치"라는 초보자의 심적 모델과 일치합니다.
