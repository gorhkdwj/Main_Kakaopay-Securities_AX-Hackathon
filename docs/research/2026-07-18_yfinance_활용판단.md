## 1. 실증으로 확정된 사실 (2026-07-18 조사)

- **거시 지표 전부 수집 가능** `[실증]`: ^KS11(코스피)·^KQ11(코스닥)·^KS200·^GSPC(S&P500), KRW=X(원/달러), ^TNX(미 10년 금리), ^VIX, GC=F(금)·CL=F(유가).
- **VKOSPI(한국 변동성지수)는 Yahoo 미제공** `[실증: 3종 티커 모두 404/빈 데이터]` → 대안: ^VIX 사용, 또는 코스피 시계열로 실현변동성 자체 계산(결정론 엔진 소관).
- **한국 대형주 데이터 양호** `[실증: 005930.KS]`: info 161개 키(섹터·산업·베타·52주 고저·시총·배당수익률·50/200일 이평), OHLCV(분봉은 최근 60일 한정), 배당 이력 59건, 연간·분기 재무제표(KRW), 실적 발표일. **없는 것**: 옵션체인·ESG·ISIN·한국어 검색·한글 뉴스.
- **시세는 20분 지연**(.KS/.KQ — Yahoo 공식 + `exchangeDataDelayedBy=20` 실증) → "실시간"으로 표기 금지, 기준시각 배지 필수.
- **rate limit(429) 이슈 상존**(비공식 스크레이핑 기반, GitHub #2125·#2422·#2518) + `earnings_dates`는 lxml 별도 필요 → **1회 수집 → JSON 동결 스냅샷 → 기준시각 배지** 경로가 유일하게 안전(헌법 §14 "실시간 API 의존 금지"와도 정합). 데모 중 라이브 호출 금지.

## 2. 제품 관점 4분류 (사용 가능/금지 판정)

### (a) 시장 맥락 카드 — W4 1차 수집 대상 ✅
종목 무관 거시 지표라 가상 종목 원칙(§7)과 충돌 없음. 전부 실증 완료.

| 지표 | 티커 | 카드 표기 예 |
|---|---|---|
| 코스피 / 코스닥 / 코스피200 | ^KS11 / ^KQ11 / ^KS200 | 전일 종가·등락률 |
| S&P500 (미 증시 맥락) | ^GSPC | 전일 종가·등락률 |
| 원/달러 환율 | KRW=X | 종가·등락률 |
| 미 10년 금리 | ^TNX | 수치(%p) |
| 변동성(미국) | ^VIX | 수치 + "한국 VKOSPI는 미제공, 참고 지표" 병기 |

### (b) 종목 팩트 카드 — 실종목 사용 시(현재 보류)
섹터·산업, 베타, 52주 고저, 시총, 거래량 vs 평균, 이동평균, 배당(수익률·이력·배당락), 확정 재무·마진·ROE, 내부자·기관 지분율. **사실 서술만** — 이평 해석(골든크로스 등)은 interpretations로 분리해야 함. ⚠️ 실종목 화면 노출은 §7(가상 종목만)과 충돌 — **Decisionlog 결정 선행 필요**. 현 데모에선 가상 fixture의 현실감 참고 자료로만 사용.

### (c) 챗봇(W2) 컨텍스트 — 시장 맥락 스냅샷 주입
(a) 스냅샷을 W2 대화 컨텍스트에 격리 주입(출처 Yahoo Finance·기준시각 병기). 숫자 계산은 엔진 전담. 사업 개요(longBusinessSummary — 영문)·재무제표·뉴스(영문)는 데모 범위에서 제외(가상 종목과 불일치 + 번역 부담).

### (d) 금지 / 주의 — 정책 경계 ❌
- **금지(§14 위반)**: `recommendations`(매수/매도 의견 집계), `analyst_price_targets`(목표주가), `upgrades_downgrades`, `info`의 `recommendation*`·`target*` 필드, WebSocket 실시간 스트리밍.
- **핵심 함정**: `info` 딕셔너리에 **권유성 필드가 사실 필드와 혼재** — 통짜 전달 금지, **화이트리스트 추출 함수 필수**(수집 스크립트 단계에서 차단).
- **주의(추정치·프레이밍)**: `earnings/revenue_estimate`·`eps_trend`·`growth_estimates`·`forwardPE`(미래 추정), `undervalued_*`/`day_gainers` 등 사전정의 스크리너(방향 프레이밍), 미래 실적 예정일(공시된 예정일임을 명시할 때만).

## 3. W4 구현 제안 (승인 대기)

1. `scripts/data/collect_market_context.py` 신규 — (a) 목록 1회 수집, **화이트리스트 필드만** 추출, `data/snapshots/market_context_YYYYMMDD_HHMM.json`으로 동결(as_of·source·지연 고지 포함).
2. 수집 실패 항목은 스냅샷에서 제외(부분 성공 허용) — 데모는 스냅샷 파일이 없으면 카드 자체를 숨기는 안전 강등.
3. 웹앱 ① 또는 ② 화면에 '시장 맥락' 카드 — 출처(Yahoo Finance)·기준시각·"지연 데이터" 배지. 가상 종목 시나리오와 분리된 시장 정보임을 명시.
4. W2 챗 컨텍스트에 동일 스냅샷 주입.
5. 계약 문서 선행 갱신: 스냅샷 스키마(§3)·표기(§9) — 구현 전 수행.
