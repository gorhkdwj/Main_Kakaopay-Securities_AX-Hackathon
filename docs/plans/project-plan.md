## 1. 문제 정의

### 1.1 최종 문제 문장 (교차검증 수렴 — X-CLM-0036, H)
> 가격 변동과 손실 때문에 판단이 흔들리는 경험 부족 투자자가 — ① 보유 종목 급락·손실 국면("지금 팔아야 하나"), ② 첫 매수 주문 순간, ③ 매도·매도 후 처리 국면에서 — [왜 이런 일이 벌어졌는지의 근거], [무엇을 확인해야 하는지의 절차], [이 선택의 비용·위험·비가역성]을 이해하고, 자신의 원래 계획·최신 사실·주문 결과를 대조하여 **스스로 설명 가능한 결정**을 내리도록 돕는 문제. 해법은 결론 대행이 아니라 **출처 있는 판단 재료의 구조화와 선택권의 반환**이다.

JTBD 한 문장(X-CLM-0009): "변동성 때문에 불안해졌을 때, 누군가의 결론을 따르는 대신 내 계획과 최신 사실, 주문 후 결과를 한 화면에서 비교하여 내가 감당할 수 있는 결정을 내리고 싶다."

**제품 정체성과 데모 전략의 관계 (D-0716-1026-main)**: 제품은 위 3국면을 포함한 **모든 매수·매도·보류 판단 순간**을 대상으로 설계한다(1층 — 상황 불문 정체성). 급락·손실 국면은 그중 판단 왜곡이 실증적으로 가장 큰 **우선 공략 지점이자 대표 데모 시나리오**다(2층 — 전략, §2.4). 실증 근거 자체가 방향 중립적이다: 처분효과는 손실 보유만이 아니라 **수익 국면의 성급한 매도**(보유 첫날 매도율 수익 41% vs 손실 22%)도 포함하며, 출제 의도도 "손해 불안"과 "매수·매도 막막함" 두 축이다.

## 2. 타깃 사용자

### 2.1 정의 원칙 — 나이·자산이 아니라 경험·역량 (H)
"20대·소액 = 초보"는 기각(X-CLM-0040). 행태 문제·저성과는 연령보다 **투자경험(신규 여부)**에서 갈린다(KCMI). 제품 내부 라벨은 단일 '초보/숙련'이 아니라 `신규 진입`·`역량 초보`·`현재 행동취약` 3필드 보존(F-CLM-0033).

### 2.2 조작적 정의 축 (요약 — 전체 표는 리서치 §2.2)
① 진입기간(첫 체결 후 6·12·24개월 민감도 — 12개월은 임시 스크리너, L) ② 주문 경험(누적 매도 5회 미만 등) ③ 객관 지식(8~12문항 — 성인 65.7점·20대 취약, H) ④ 자기효능감(4~6문항 — **과잉확신형**은 저효능형과 다른 메시지) ⑤ 현재 행동취약 상태(불안·예상후회·현금 필요·계획 변경 이유를 **별도 질문** — 행동 신호로 심리를 진단하지 않음) ⑥ 정보원 의존·행동 신호(처벌적 라벨 금지).

### 2.3 우선 세그먼트 (모두 "우선 검증 가설" M — 확정 사실 아님)

## 3. 솔루션 개요 — 판단 여권

### 3.1 8단계 사용자 흐름 (리서치 §9.2 확정안 — D-07 재판정 반영)
① 상황 선택·계획 회상(계획 없으면 결론 대신 질문 초안) → ② 관련 사실 브리핑(출처·기준시각, `확인된 사실/해석/알 수 없음` 분리, 뉴스 원천 없으면 '원인' 미출력) → ③ 판단 체크리스트 5개+이해 확인 → ④ 대칭 시나리오(유지/일부/전량 — 기본 강조 없음)+주문 영향 미리보기(수수료·세금·실현손익·잔여비중·D+2·체결시장) → ⑤ 검토 의향 4택+이유 기록(주문 CTA 아님, **추가매수는 요청 시만**) → ⑥ 실행 분리(별도 화면 재확인→**모의 체결**) → ⑦ 사후 기록(과정의 일관성 회고 — 모의 체결 시 "실제 주문 화면으로" 연결, 보류 시 처음으로) → ⑧ 주문 화면(실앱 재현·비기능 — 실서비스에서 기존 주문 모듈로 핸드오프되는 목적지의 시연, 홈에서 "브리핑 없이 바로 주문" 직행도 가능 — 사용자 결정 2026-07-16). 장애 안전모드는 위저드 단계가 아니라 **DEMO 패널 토글의 전 화면 오버레이**(VOC 최상위 불만 대응, D-08)로 어느 화면에서든 전환된다.

**확장(D-0716-1352-main)**: ①에 **지난 투자 일지 리마인드** 추가 — 브리핑 모드 사용자는 첫 매수부터 매 거래에서 이유를 기록하므로, 기존 보유자의 판단 시작 시점에는 '매수 당시의 일지'가 자연히 존재한다(제품 논리). 계획 카드와 나란히 지난 일지를 대조 표시한다. 예외 2가지를 병기: 간단 모드 사용자는 일지 없음(리마인드 없음 — 행동취약 신호 시 1탭 재제안만), 일지 작성은 권장이되 강제 아님(빈 기록 허용 — 강요 금지).

**추가 확장(D-0716-1346-main)**: 탐색 테마 경유 구매 시 **투자 일지 자동완성 초안** 제공 — 진입 맥락(테마명·기준·일시)의 행동 사실만으로 초안을 조립하고(시스템이 감정·판단을 창작하지 않음), 사용자가 수정·채택한다(자동 저장 금지 — "스스로 설명 가능한 결정"의 인지 가치 보존). 명칭 확정: 화면 명칭은 **"투자 일지"**(내부 스키마명 decision_record는 유지).

### 3.2 화면 상수 4종
무엇을 아는가(출처·기준시각) / 무엇을 모르는가 / 무엇이 달라지는가(현금·비중·비용·결제일·체결시장) / **누가 결정하는가(사용자)**.

### 3.4 차별화 근거 (실기기 확인으로 갱신 — 2026-07-16, D-0716-1310-main)
- **실기기 확인**: 카카오페이증권 앱 캡처 24장 분석(`docs/references/image/`) 결과, 실앱은 이미 ①국내 종목 **AI 급등락 요약**("AI분석" 배지+기준일) ②주문 확인 시 예상 수수료 표시 ③사후 실현손익 달력 ④용어 교육 툴팁을 제공한다. **따라서 차별점을 "요약·안내의 유무"로 말하지 않는다**(리서치 F-CLM-0010의 '공개 조사 미확인'은 실기기 확인으로 대체).
- **확인된 공백(캡처로 확증)**: 매도 확인 화면에 거래세·농특세/실현손익 **사전** 미리보기/D+2 출금일/비가역성 고지가 없고(수수료만 — "총 주문 금액에 포함되지 않아요"), AI 요약에 출처·기준시각·양면 해석·'알 수 없음'이 없으며, 계획 대조·판단 이유 기록·회고가 없다.
- **차별점 문장**: 판단 여권은 이 공백 — **판단 재료의 구조화(출처·양면·모름) + 비가역 결정 전 비용·정산 사전 고지 + 과정 기록** — 을 기존 앱의 UX 언어 위에 얹는 레이어다([S4 화면 스펙](s4-ui-spec.md) §2). 결합 원형: Betterment Tax Impact Preview + 토스 AI 맥락 요약 + Vanguard 3A 행동 코칭 + 출처·기준시각 표기.
