## 1. 문제 정의

### 1.1 최종 문제 문장 (교차검증 수렴 — X-CLM-0036, H)
> 가격 변동과 손실 때문에 판단이 흔들리는 경험 부족 투자자가 — ① 보유 종목 급락·손실 국면("지금 팔아야 하나"), ② 첫 매수 주문 순간, ③ 매도·매도 후 처리 국면에서 — [왜 이런 일이 벌어졌는지의 근거], [무엇을 확인해야 하는지의 절차], [이 선택의 비용·위험·비가역성]을 이해하고, 자신의 원래 계획·최신 사실·주문 결과를 대조하여 **스스로 설명 가능한 결정**을 내리도록 돕는 문제. 해법은 결론 대행이 아니라 **출처 있는 판단 재료의 구조화와 선택권의 반환**이다.

JTBD 한 문장(X-CLM-0009): "변동성 때문에 불안해졌을 때, 누군가의 결론을 따르는 대신 내 계획과 최신 사실, 주문 후 결과를 한 화면에서 비교하여 내가 감당할 수 있는 결정을 내리고 싶다."

**제품 정체성과 데모 전략의 관계 (D-0716-1026-main)**: 제품은 위 3국면을 포함한 **모든 매수·매도·보류 판단 순간**을 대상으로 설계한다(1층 — 상황 불문 정체성). 급락·손실 국면은 그중 판단 왜곡이 실증적으로 가장 큰 **우선 공략 지점이자 대표 데모 시나리오**다(2층 — 전략, §2.4). 실증 근거 자체가 방향 중립적이다: 처분효과는 손실 보유만이 아니라 **수익 국면의 성급한 매도**(보유 첫날 매도율 수익 41% vs 손실 22%)도 포함하며, 출제 의도도 "손해 불안"과 "매수·매도 막막함" 두 축이다.

### 1.2 문제가 실재한다는 근거 3계층 (수준 구분 필수)
| 계층 | 내용 | 한계 |
|---|---|---|
| 계좌 실증 (F, H) | 신규 투자자 비용 차감 후 수익률 **-1.2%** vs 기존 **+15.0%**, 손실 계좌 60% vs 36%(KCMI, 2020.3~10, 표본 204,004명). 보유 첫날 매도율 수익 41% vs 손실 22%(=1.86배), 처분효과는 신규에서 더 강함. Odean PGR/PLR 0.148/0.098(=**1.51배** — 국내 수치와 혼용 금지, CV-02) | 2020 강세장·미국 1987~96 표본 — 현재 고객 직접 일반화 금지. **행태 관찰은 심리 원인 진단이 아님**(F-CLM-0115) |
| VOC (M) | 독립 두 표본(121건+118건)이 동일 테마 수렴: ① 결정적 순간의 배신(장애로 매도 실기·의도와 다른 체결) ② 위험·비용 사전 고지 공백(모르는 새 미수·자동환전) ③ 요구는 결론 대행이 아닌 **판단 재료의 표시·설명** | 자기선택 표본 — 비율·우열 일반화 금지(F-CLM-0047) |
| 출제 의도 (F) | 인터뷰 원문: "내가 손해 보지 않았나"(손실 불안) + "어떻게 해야 될지 모르겠다"(막막함). 평가는 "수학적 최적점"이 아니라 "납득·안심·실행"과 "과정의 설계" | 문제의 존재 언명이지 유병률 아님 |

**따라서 이 문제정의는 근거가 강한 제품 가설(M)이며**, 발표·브리프에서 "검증된"이라 말하지 않고 "근거 있는 가설 + 검증 계획"으로 말한다(§7 발표 원칙).

### 1.3 문제에서 제외하는 것
실거래 체결, 수익 보장, 확정적 방향 추천, 자율 주문, 개인화 종목 추천(규제 🔴 + 효과 근거 부족으로 기각 X-CLM-0041), 최적 매매시점 예측.

## 2. 타깃 사용자

### 2.1 정의 원칙 — 나이·자산이 아니라 경험·역량 (H)
"20대·소액 = 초보"는 기각(X-CLM-0040). 행태 문제·저성과는 연령보다 **투자경험(신규 여부)**에서 갈린다(KCMI). 제품 내부 라벨은 단일 '초보/숙련'이 아니라 `신규 진입`·`역량 초보`·`현재 행동취약` 3필드 보존(F-CLM-0033).

### 2.2 조작적 정의 축 (요약 — 전체 표는 리서치 §2.2)
① 진입기간(첫 체결 후 6·12·24개월 민감도 — 12개월은 임시 스크리너, L) ② 주문 경험(누적 매도 5회 미만 등) ③ 객관 지식(8~12문항 — 성인 65.7점·20대 취약, H) ④ 자기효능감(4~6문항 — **과잉확신형**은 저효능형과 다른 메시지) ⑤ 현재 행동취약 상태(불안·예상후회·현금 필요·계획 변경 이유를 **별도 질문** — 행동 신호로 심리를 진단하지 않음) ⑥ 정보원 의존·행동 신호(처벌적 라벨 금지).

### 2.3 우선 세그먼트 (모두 "우선 검증 가설" M — 확정 사실 아님)
- **핵심**: 보유 중 손실불안형 + 매도 판단 막막형(하나의 연속 여정) — 핵심 데모의 타깃(F-CLM-0036·0037).
- **포함**: 첫 매수 준비형 — 백업 데모(K2)의 타깃.
- **백로그**: 매도 후 재진입형(국내 직접 근거 부족 — 양측 독립 판정 일치로 후순위 확정, C-CLM-0217).

### 2.4 대표 페르소나·시나리오 (데모 기준 — [목업](../mockup/2026-07-17_판단여권_목업.html) S0~S8)
"지민": 첫 체결 8개월차, 3종목 중 2종목 손실, 누적 매도 2회, 지식 퀴즈 하위권. **서킷브레이커 발동일에 보유 '가온전자(가상)' -8% + 실적 악재**를 마주함. 시의성 근거: 본선 직전 실제로 서킷브레이커 안내 3회(06-26·07-07·07-13)+앱 접속 지연 장애(06-30) 발생(C-CLM-0106, H).

## 3. 솔루션 개요 — 판단 여권

### 3.1 8단계 사용자 흐름 (리서치 §9.2 확정안 — D-07 재판정 반영)
① 상황 선택·계획 회상(계획 없으면 결론 대신 질문 초안) → ② 관련 사실 브리핑(출처·기준시각, `확인된 사실/해석/알 수 없음` 분리, 뉴스 원천 없으면 '원인' 미출력) → ③ 판단 체크리스트 5개+이해 확인 → ④ 대칭 시나리오(유지/일부/전량 — 기본 강조 없음)+주문 영향 미리보기(수수료·세금·실현손익·잔여비중·D+2·체결시장) → ⑤ 검토 의향 4택+이유 기록(주문 CTA 아님, **추가매수는 요청 시만**) → ⑥ 실행 분리(별도 화면 재확인→**모의 체결**) → ⑦ 사후 기록(과정의 일관성 회고) → ⑧ 장애 안전모드(엔딩 신 — VOC 최상위 불만 대응, D-08).

### 3.2 화면 상수 4종
무엇을 아는가(출처·기준시각) / 무엇을 모르는가 / 무엇이 달라지는가(현금·비중·비용·결제일·체결시장) / **누가 결정하는가(사용자)**.

### 3.3 하지 않는 것 (헌법 §14와 동일 — 변경 시 Decisionlog)
개인화 종목 추천 · 방향 결론 · AI 자동·실주문 · 대화 내 주문 실행 버튼 · 수익 예측·보장 · 게임화 · 실시간 API 의존 · 손실 화면 추가매수 기본 노출.

### 3.4 차별화 근거 (기회 가설)
매수 UX는 상향평준화된 반면, 공개 조사에서 "등락 관련 사실+출처·기준시각+매도 전 비용·정산"을 한 흐름으로 묶은 국내 사례를 확인하지 못함(부재 증명 아님 — F-CLM-0010, M). 결합 원형: Betterment Tax Impact Preview + 토스 AI 맥락 요약 + Vanguard 3A 행동 코칭 + 출처·기준시각 표기.

### 3.5 온보딩·모드(구현 후순위 — [Decisionlog D-0716-0210-main](../../Decisionlog.md))
간단 모드/브리핑 모드 선택, 옵트아웃 불가 최소선(비용·세금·D+2·비가역 고지), 행동취약 신호 시 1탭 재제안. 본선 데모 범위 외 — 심사 Q&A 답변만 지참.

### 4.2 북극성 지표 (설계 목표 — F-CLM-0107, M)
"핵심 조건을 이해하고, **사전 계획을 따르거나 새 근거로 변경 이유를 명시한 뒤**, 거래·보류 중 감당 가능한 결정을 완료한 세션 비율" — 보류도 성공. 거래 횟수·체류시간·단기수익률은 성공지표로 쓰지 않는다(과잉거래 실증 11.4% vs 17.9% + 게임화 제재 기록이 근거).

### 4.3 실서비스 가정 지표 (가설 L — 파일럿 검증 대상)
이해(주문유형·체결·비용·정산 4문항 전후 비교) / 판단 품질(계획-선택 일치율·근거 1개 이상 설명 가능 비율) / 불안·자기효능(전후 변화 — 불안 0이 목표 아님) / 안전(금지 권유 출력 0 — 출시 게이트) / 고객(정산·평단 문의 셀프서비스 해결률·재문의율) / 비즈니스(30·90일 건강한 재방문 — **과잉거래·고위험 집중 증가를 중단 기준으로 감시**). 검증 단계: 문제 인터뷰 8~12명 → 사용성 5명 반복 → 오프라인 AI 평가 → 모의투자 파일럿 → 준법 승인 후 제한 A/B.

- **실주문·방향 추천을 명시 요구** → 규제 경계(리서치 §7)를 근거로 "모의 체결+결론 비제시" 대체안을 제안하고 담당자 확인(당일 공지 우선).
- 발표 뼈대: 문제 정의 → 타깃·페인포인트 → AI 적용 지점과 안전장치 → 검증 — "과정의 설계"·"스스로 증명" 심사 기준 정렬. 이중 리서치·교차검증(오류 20건 자체 발견·정정)은 과정 증명의 소재이되, **제품 효과의 증명은 아니라고 명시**한다.
