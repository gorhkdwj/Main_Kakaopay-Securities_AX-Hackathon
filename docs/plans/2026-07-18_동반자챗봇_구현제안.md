# 동반자 챗봇 통합 구현 제안서 (전역 오버레이 방식)

> **지위**: 구현 제안(참고용) — 사용자 승인된 방향("전역 플로팅 버튼 + 오버레이 패널", 2026-07-18 대화)을 병렬 구현 세션이 참고할 수 있게 정리한 문서. 계약(`docs/requirements-contract.md`)과 충돌하면 계약이 우선한다.
> **대화 설계 원본**: `docs/mockup/2026-07-18_동반자챗봇_목업.html` — 문답 구조·카드 문법·가드 라벨·핸드오프 패턴은 이 목업을 그대로 따른다.
> **작성**: main 세션 (W-0718-1140-main). 구현 소유권: 구현 세션(src/webapp — §13-5).

## 0. 결정 요약 (왜 전역인가)

| 선택지 | 판정 | 이유 |
|---|---|---|
| **A. 전역 플로팅 버튼 + 풀높이 시트(채택)** | ✅ | ① "동반자" 서사 — 어느 단계에서든 부를 수 있어야 성립, 특히 ④~⑥(마지막 실행 단계)에서 필요 ② 기존 검증된 9개 화면 DOM 무수정(독립 레이어 — 회귀 위험 최소) ③ 오버레이 인프라 3종 기존재(인터셉트·⑥시트·안전모드) ④ 시연 한 문장: "어느 화면에서든 이 버튼" |
| B. 특정 페이지 삽입(②·④ 등) | ❌ | 해당 renderStepN 수정 → 통과된 화면 검증 재개봉. "동반자" 서사 반쪽 |
| C. 화면 사이드 패널 | ❌ | 폰 프레임 430px에 사이드 공간 없음. 모바일 관례(플로팅 버튼→풀시트)에서 이탈 |

보완: ② 브리핑 하단에 인라인 진입점 1개("더 궁금한 건 동반자에게 →" — 같은 패널 열기)로 발견성 확보. (선택 구현 — 시간 없으면 생략 가능)

## 1. UI/DOM 계약

```html
<!-- index.html — #phone 안, 기존 오버레이들과 형제 레벨 -->
<button id="companion-fab" type="button" aria-label="동반자와 대화">◆</button>

<div id="companion-backdrop" hidden>            <!-- 배경 딤 + 탭 시 닫기 -->
  <section id="companion-panel" role="dialog" aria-modal="true" aria-label="동반자 대화">
    <header>동반자 <span class="pill">판단 여권 AI</span>
      <span id="companion-asof"><!-- 기준시각·출처 배지 --></span>
      <button id="companion-close" type="button" aria-label="닫기">✕</button>
    </header>
    <div id="companion-log"><!-- 말풍선 누적 --></div>
    <div id="companion-quick"><!-- 빠른 질문 칩 --></div>
    <form id="companion-form"><input id="companion-input"><button type="submit">↑</button></form>
  </section>
</div>
```

- **표시 토글**: `hidden` 속성만 사용 — 전역 `[hidden]{display:none!important}` 규칙이 이미 있음(T-0716-1510). 별도 display 조작 금지.
- **z-index**: companion-backdrop < #sheet-backdrop·#intercept-backdrop(기존 팝업이 항상 위). **⑥ 시트/인터셉트가 열려 있는 동안 FAB 숨김**(`#sheet-backdrop:not([hidden]) ~ #companion-fab { display:none }` 또는 JS 동기화) — 배타 규칙으로 겹침 사고 차단.
- **안전모드**: 안전모드 오버레이 진입 시 FAB·패널 모두 숨김(대화도 판단 재료 공급원 — 장애 시 함께 강등). 기존 안전모드 토글 핸들러에 1줄 추가.
- **클래스 명명 주의**: `kp-dot`은 탭 알림 점으로 사용 중 — 접두사 `cmp-`(companion) 권장. 금지 마커 `data-role="order-execute"`·"체결하기" 문구를 패널 안에 절대 넣지 않는다(계약 §9).
- **패널 형태**: 폰 프레임 안 하단에서 올라오는 풀높이(약 92%) 시트, 전 모서리 radius 18px, 등장 모션은 ⑥ 시트 모션 재사용.

## 2. 대화 컨텍스트 주입 (티키타카의 핵심)

패널 열기·질문 전송 시 클라이언트가 함께 보내는 컨텍스트:

```js
{
  scenario_id: S.scenarioId,
  step: S.step,                    // "지금 ④ 비교 화면을 보고 계시네요" 응답 가능
  flow_side: S.flowSide,           // buy | sell
  active_calculation_id: (현재 화면의 preview?.calculation_id ?? null),
  question: "<사용자 입력>",
  history: [ {role, text} ... ]    // 이 세션 대화 이력(클라 보관, 서버 무상태)
}
```

## 3. 백엔드 API 제안

`POST /api/companion/chat` (app.py 신설)

- **응답 계약**: 브리핑과 동일한 AI 응답 계약(§6) 재사용 — facts(+source_id·as_of) / interpretations(양면) / unknowns / next_questions + `reply_text`(말풍선용 서술). **렌더 전 guard 통과 필수**(기존 `src/policy` 그대로).
- **데이터 원천**: 실종목 시나리오(§3.1-b)의 fixture + `data/snapshots/market_context_*.json` — **LLM 프롬프트에는 §3.1-b 화이트리스트 필드만**(권유성 필드 금지 규칙 승계). 숫자 계산(분할 비교·수수료·가격 위치)은 **엔진 함수 호출 후 결과만 프롬프트에 주입**(산수 금지 — llm.py `build_messages` 패턴 재사용, `_strip_decorative` 통과).
- **폴백 사슬**(§8과 동일 철학): live 실패·타임아웃 → **사전 준비 문답 캐시**(아래 §4) → "지금은 답변을 준비할 수 없어요 — 화면의 사실 카드를 참고해 주세요"(안전 강등 문구). 어떤 실패도 silent 금지(T-0716-2046).

## 4. 시간 역산 구현 순서 (13:00 마감 기준)

| 단계 | 내용 | 없으면 |
|---|---|---|
| **MVP-1 (필수)** | FAB + 패널 + 목업의 4문답을 **사전 준비 캐시**(JSON — 질문 매칭은 quick 칩 고정)로 재생: 칩 탭 → 캐시 응답 렌더. LLM 호출 0회로도 시연 성립 | 챗봇 자체가 없음 |
| MVP-2 | 자유 입력 → live LLM(`call_anthropic` 재사용, 타임아웃 30초) + guard + 캐시 폴백 | 칩 고정 문답만으로 시연 |
| 선택 | ② 인라인 진입점 · step 인지 인사말 · 대화 이력 저장 | 생략 무방 |

- 캐시 파일 위치 제안: `data/fixtures/companion_cache/real_005930.json` (기존 llm_cache 규약처럼 생성 스크립트+지문은 과함 — 시간상 수기 검수 캐시 허용, `[데모 고정]` 표기).

## 5. 안전·검증 체크리스트 (구현 완료 조건)

1. 패널 텍스트 전부 lexicon 통과 — **app.js·index.html에 넣는 순간 `test_static_files_pass_lexicon` 자동 검사 대상**이 된다. 특히: "원인" 단어 금지(test_no_cause_word — 목업의 "왜를 확정할 수 없어요" 프레임 사용), 추가 매수 유도 표현 금지, "최고/최저"는 검사 통과 확인됨(차트 주석 선례).
2. 패널 안 주문 실행 버튼 0 · `data-role="order-execute"` 0 · "체결하기" 문구 0(기존 DOM 테스트가 정적 파일 전체를 스캔하므로 자동 검증됨).
3. 모든 사실 카드에 source_id(YF-SRC-### — §3.1-b)·as_of 배지, 패널 상단에 "기준시각 ··· · 지연 시세 고지" 상시.
4. 안전모드 토글 → FAB 숨김 왕복 · ⑥ 시트 열림 중 FAB 숨김 — playwright 확인.
5. pytest 전체 + 게이트 + 실브라우저 완주 1회(S0→⑧ 기존 흐름이 패널 추가로 깨지지 않음 — 오버레이 독립성 확인).

## 6. 재사용 자산 지도

| 필요한 것 | 이미 있는 것 |
|---|---|
| 오버레이 열기/닫기·모션 | `#sheet-backdrop`·`openSheet()`(app.js) 패턴 |
| LLM 호출·프롬프트 격리·타임아웃 | `src/briefing/llm.py` — `build_messages`(`<data>` 인젝션 격리)·`call_anthropic`(30초)·`_strip_decorative` |
| 응답 차단 | `src/policy` guard·lexicon 그대로 |
| 카드 문법(사실/양면/모름/계산 배지) | 목업 HTML의 `.tag.fact/.pos/.neg/.unk/.calc` + 본체 app.css의 동명 계열 토큰 |
| 수수료·분할 비교 숫자 | `src/engine` (분할 비교 함수가 없으면: 1회분 = buy_preview 재사용 × 횟수 — 신규 산식 도입 전 계약 §5 확인) |
| 문답 대본 | 목업 4문답(타이밍 불안 / 하락 사유 / 분할 비교 / 결정 핸드오프) — 문구 재사용 시 lexicon 재확인만 |
