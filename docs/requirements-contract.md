# 기준 계약 문서 · 판단 여권 (Decision Passport)

> 구현 전에 합의해 고정한다. 기준이 바뀌면 코드보다 이 문서를 먼저 갱신하고 Decisionlog에 이유를 남긴다. 같은 지표가 파일마다 다르게 정의되지 않게 한다. `확인 필요` 항목은 구현 S1에서 확정한다.

## 입력 형식
- **스냅샷**: `data/snapshots/snapshot_YYYYMMDD.json(csv)` — 종목코드·종목명(가상)·종가·전일대비·거래량·20일 평균 거래량. `manifest.json`에 수집 시각(KST)·출처·성공 여부·SHA-256 기록. 수집 실패 시 해당 스냅샷은 사용 금지.
- **fixture**: `data/fixtures/scenario_{first_buy|loss8|profit15}.json` — 가상 종목·보유(수량·평단)·공시 요지(+가상 출처 라벨·발행시각)·계획(목표기간·감수손실·재검토 조건).
- **사용자 입력**: 상황 선택(5택), 주문 수량(정수, 1~보유수량), 검토 의향(4택), 이유 텍스트(선택), 이해 확인 응답(O/X).

## 출력 형식
- **AI 응답 계약(JSON)** — 모든 LLM 출력의 유일한 형식:
  `facts[{text, source_id, as_of}] / interpretations[{text, basis[]}] / unknowns[] / user_inputs{} / calculation_id / policy_result / next_questions[]`
- **계산 결과(JSON)**: `calculation_id, 입력(수량·기준가), 매도대금, 수수료, 거래세, 예상수령액, 실현손익, 잔여수량, 잔여비중(전/후), 결제일(D+2 영업일), as_of`.
- 화면 표기: 금액은 원 단위 콤마, 비중은 소수 1자리 %, 손익은 색+부호(±)+아이콘(▲▼)+텍스트 병행.

## 주요 지표 정의
- **잔여 비중** = 매도 후 해당 종목 평가액 ÷ 총 평가자산(스냅샷 기준) × 100.
- **실현손익** = 예상수령액 − (평균 매수가 × 매도수량). (평가손익과 구분해 표기)
- **안전 지표**: 출처 없는 사실 수 / 금지 표현 수 / 기준시각 누락 수 — 분모는 "해당 세션에서 렌더링된 사실 문장 수".

## 수식 (데모 기준값 — 국내주식 예시, 시장·상품별 규칙 엔진으로 확장)
- 수수료 = 매도대금 × 0.015%(KRX) / 0.014%(NXT) — `확인 필요: 데모는 KRX 단일로 고정할지 S1에서 결정`
- 거래세 = 매도대금 × 0.20% (국내주식 매도 예시 — 시장·상품 일반화 금지)
- 예상수령액 = 매도대금 − 수수료 − 거래세
- 결제일 = 체결일 + 2영업일(토·일 제외, 공휴일 캘린더는 데모 범위에서 주말만 처리 — 한계 명시)
- 반올림: 수수료·세금 원 미만 절사 `확인 필요: 실제 규칙 원문 재확인, 데모는 절사로 고정`

## 상태 판정 기준
- `facts`의 모든 항목은 source_id·as_of 필수 — 없으면 렌더링 차단.
- 데이터 기준시각이 스냅샷과 불일치하면 `확인 불가` 상태 카드 출력(지어내지 않음).
- 장애 모드 진입: 수동 토글(데모) — 진입 시 신규 주문 유도 UI 숨김, 주문 상태·공지만 표시.

## 오류 판정 기준 (정책 가드 — 렌더링 전 차단)
- 금지 표현: 방향 결론("사세요/파세요/보류하세요"), 단정("오릅니다/안전합니다/확실"), 수익 보장·예측 수치.
- 사용자 입력과 다른 수량 제안, 미래 수익률 숫자, source_id 없는 외부 사실.
- 과대 주문: 수량 > 보유수량 → 오류, 주문 금액이 총자산의 일정 비율 초과 시 경고 `확인 필요: 경고 임계값`.

## 허용 값 / 허용하지 않을 값
- 허용: 가상 종목 데이터, 동결 스냅샷, 사용자 입력 수량 기반 계산, 감정 인정 문구("불안하실 수 있습니다")까지.
- 비허용: 실존 종목 실데이터로 방향성 있는 예시, 실주문 API 호출, 개인화 추천, 손실 화면의 추가매수 기본 노출, 대화 내 주문 실행 버튼.

## 누락 데이터 처리
- 스냅샷 필드 누락 → 해당 카드 `확인 불가` 표시(대체값 생성 금지). 공시 없음 → "확인된 공시 없음"을 사실로 표기.

## 시간대·단위·반올림 기준
- 모든 시각 KST, `as_of`는 `YYYY-MM-DD HH:mm KST` 표기. 금액 KRW 원 단위. 비중 소수 1자리.

## 완료 기준 (데모)
1. -8% 시나리오 8단계를 오프라인(fixture)으로 무중단 완주.
2. 안전 게이트: 사전 정의 테스트셋(validation-plan)에서 출처 없는 사실 0·금지 표현 0·기준시각 누락 0.
3. 계산 엔진 단위 테스트 전건 통과(수수료·세금·D+2·비중 — 경계값 포함).
4. 브리프 4항목과 시연 흐름이 로그·구현과 모순 없음.
