# 검증 계획 · 판단 여권 (Decision Passport)

기능을 만들 때마다 영향 범위에 맞게 검증한다. 검증하지 못한 부분은 숨기지 않고 `미검증 범위`에 적는다. 실패한 테스트를 삭제·완화하지 않고 원인을 수정한다. (심사 대응: "어떻게 검증했나"의 답이 이 문서다.)

## 검증 항목
- **단위 테스트**: 계산 엔진(수수료·거래세·수령액·실현손익·비중·D+2) — 정상/경계(0·1·전량)/오류(초과 수량) 케이스. 기준값은 requirements-contract 수식.
- **정책 가드 테스트(안전 테스트셋)**: 금지 표현(방향 결론·단정·보장), 무출처 사실, 수량 불일치, 미래 수익률 — **차단되어야 할 입력 목록과 통과해야 할 정상 입력 목록을 사전 정의**하고 분모를 기록한다(예: 차단 케이스 N건 중 N건 차단, 정상 케이스 M건 중 오차단 0건). 프롬프트 인젝션 케이스(공시 텍스트 안 지시문) 포함.
- **통합 테스트**: fixture 3종 각각으로 8단계 완주(자동화 가능 범위) + 수동 리허설 체크리스트.
- **오류 상황 검증**: 스냅샷 필드 누락→`확인 불가` 카드, LLM 타임아웃→응답 캐시 폴백, 장애 모드 토글→주문 유도 숨김.
- **오프라인 검증**: Wi-Fi 차단 상태에서 -8% 시나리오 완주 1회(7/17 밤, 당일 아침 재확인).
- **문서-코드 정합성**: 계약 문서 수식 = 코드 상수 = 화면 표기 일치. README-기능 일치(구현된 것만 기재).
- **제출물 구조**: 브리프 4항목 ↔ 시연 흐름 ↔ 로그 정합, (형식 지정 시) 플러그인 실행 확인, 로그 수집 절차(worktree 전체) 점검.

## 검증 방법과 도구
- pytest(엔진·가드), 스키마 검증 스크립트(fixture·manifest), 수동 리허설 체크리스트(docs/references/에 기록), 안전 지표 카운터의 화면 노출(시연 중 실시간 증명).

## 안전 게이트 (시연·제출 전 필수 통과)
| 지표 | 목표 | 분모 정의 |
|---|---|---|
| 출처 없는 사실 | 0건 | 세션 중 렌더링된 facts 문장 전수 |
| 금지(권유·단정) 표현 | 0건 | LLM 출력 전수(가드 로그 기준) |
| 기준시각 누락 | 0건 | 수치·사실 카드 전수 |
| 위험 4종 고지(비용·세금·D+2·비가역) | 4/4 | 매도 흐름 1회 기준 |

## 미검증 범위 (정직 고지 — 계승된 한계)
- 실사용자 과업 테스트 미수행(리서치 한계 계승) — 효과 주장(불안 감소 등)은 가설(L).
- 공휴일 캘린더 미반영(D+2는 주말만 처리) — 데모 한계로 화면에 명시하지 않되 질문 시 답변.
- KIS 실연동·실주문 경로 — 범위 외(모의 체결만).
- 실기기 앱 비교(WS4 체크리스트)는 사용자 수행 항목으로 잔존.
