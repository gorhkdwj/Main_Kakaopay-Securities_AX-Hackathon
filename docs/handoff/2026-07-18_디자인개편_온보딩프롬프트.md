## 0. 임무 (한 줄)

판단 여권 로컬 웹앱 데모의 **비주얼 디자인을 전면 개편**한다. **Claude Design 도구**로 새 디자인을 생성하고, 그 결과를 이 프로젝트의 제약(오프라인 자기완결·제품 경계·접근성)에 맞게 **`app.css` 중심으로 반영**한다. **판단 흐름·기능·로직·계약값·DOM 요소 id·안전 장치는 그대로 보존한다(디자인만 바꾼다).** 원본 데모는 절대 깨뜨리지 않는다.

## 1. 시작 전에 반드시 직접 읽을 것

1. `CLAUDE.md` — 작업 헌법(특히 §5 계약 우선, §14 제품 경계). **이 프로젝트의 최상위 규칙이다.**
2. `docs/plans/s4-ui-spec.md` — 각 화면(S0~⑧)의 UI 규정·근거·실앱 대비 차별점.
3. `docs/mockup/2026-07-17_판단여권_목업.html` — 기존 디자인 참고.
4. `src/webapp/static/index.html` · `app.css` · `app.js` — 개편 대상 3파일. 구조를 먼저 파악.
5. **`docs/references/image/` 전체(26장 캡처)** — 실제 카카오페이증권 앱 화면. **디자인 레퍼런스다(§6 참조).** ⚠️ **실계좌 정보 포함 — 로컬 참고 전용, 커밋·푸시 금지.**

## 2. 프로젝트가 무엇인가 (맥락)

- **판단 여권(Decision Passport)**: 초보 투자자가 매수·매도·보류 판단 순간에 **출처 있는 재료**를 보고 **스스로 설명 가능한 결정**을 내리도록 돕는 데모. **AI는 결론(사라/팔라)을 말하지 않는다.**
- 해커톤(AX 인재전쟁 본선, 카카오페이증권 트랙) 데모. 심사는 **1대1 노트북 라이브 시연** + 결과물 브리프 + 프롬프트 로그.
- **데모 흐름(8단계 위저드, 단일 페이지)**: S0(실앱 주문 화면 재현 + 진입 팝업) → ① 종목·계획 → ② AI 브리핑(사실/해석/모름) → ③ 체크리스트 → ④ 시나리오 비교 → ⑤ 검토 의향 → ⑥ 모의 주문(**중앙 확인 모달 + 비가역 확인 체크**) → ⑦ 회고(투자 일지) → ⑧ 주문 화면 재현(종착). 부가: 장애 안전모드(오버레이), DEMO 패널(우하단 — 시나리오 전환·안전모드 토글).
- **스택**: FastAPI(백엔드) + **순수 vanilla JS/HTML/CSS**(프런트, 프레임워크·빌드 도구 없음). 화면 전환은 `app.js`가 DOM을 토글.

## 3. 실행·검증 방법

```
# 실행(프로젝트 루트에서)
.\.venv\Scripts\python.exe -m uvicorn src.webapp.app:app --port 9000 --reload
# → http://127.0.0.1:9000
```
- ⚠️ 포트 **8765는 이 PC에서 OS 예약 범위(8687–8786)에 걸려 기동 실패**한다 → `--port 9000`(또는 9200 등 범위 밖) 사용(T-0717-2031).
- 정적 파일(`src/webapp/static/*`) 변경은 **서버 재시작 불요** — 브라우저 **Ctrl+F5**만.
- 테스트: `.\.venv\Scripts\python.exe -m pytest -q` (302건 — 통과 유지해야 함).
- 안전 게이트: `.\.venv\Scripts\python.exe scripts\gate\run_gate.py` (통과 유지).

## 4. 어디를 만지고 어디를 만지지 않는가

**개편 대상(자유)**
- `src/webapp/static/app.css` — **전면 리스타일 자유**: 색·타이포·간격·라운드·그림자·컴포넌트 룩·모션(과하지 않게)·반응형.

**신중히(디자인 위해 필요한 만큼만)**
- `src/webapp/static/index.html` — 새 래퍼·클래스 **추가**는 자유. 단 **기존 element `id`는 절대 제거·변경 금지**(app.js가 id로 접근하고, 테스트가 id·구조를 검사한다). 마크업 재배치가 필요하면 id를 유지한 채로.
- `src/webapp/static/app.js` — **렌더 마크업 문자열**만 디자인상 필요 시 조정(클래스 추가 등). **상태·흐름·API 호출·계약 표기·id 참조는 유지.** 로직은 건드리지 않는다.

**절대 금지(건드리지 말 것)**
- `src/engine/`(계산 엔진) · `src/policy/`(가드·금지어 `lexicon.py`) · `src/briefing/` · `src/webapp/app.py`의 로직 · `docs/requirements-contract.md`(골든값·수식·고지·안전 지표) · `tests/`(기대값) · `tools/`(로그 훅 — 헌법상 수정 금지) · `logs/` · `.env` · 운영 로그(Worklog/Decisionlog/Troubleshootinglog — append-only).

## 5. 불가침 디자인 제약 (제품 정체성·규정 — 어기면 무효/실격)

1. **오프라인 자기완결**: 외부 **CDN·웹폰트(Google Fonts 등)·원격 이미지·아이콘 폰트 금지**. 모든 CSS/에셋은 **로컬·인라인·시스템 폰트·data URI**만. 현재 favicon=`data:,`, 폰트=시스템 스택. **네트워크 요청 0**을 유지한다(오프라인 완주가 데모 필수 조건).
2. **§14 제품 경계**: 방향 결론·투자 권유·수익 예측·**게임화(콘페티·축하 애니메이션·뱃지·뽑기) 금지**. 손실 화면에 추가매수를 기본 강조하지 않는다. 톤은 **차분·중립·신뢰**.
3. **접근성**: 손익 등 상태는 **색 + 부호(±) + 아이콘(▲▼) + 텍스트("수익/손실") 4중 병행**. **색만으로 정보 전달 금지**(색각이상 대응). 국내 관례색(상승 적/하락 청) 유지.
4. **상시 표기 보존(문구·존재를 없애지 말 것 — 계약·게이트가 검사)**: 상단 "모의 주문 — 실제 거래 아님" 바, "교육용 가상 데이터" 배지, **안전 지표 카운터**(렌더 사실·무출처 차단·금지 표현·기준시각 누락), 각 사실의 **출처·기준시각 배지**, ⑥의 **4종 고지**(수수료·세금/실현손익/D+2/비가역), ⑥의 **비가역 확인 체크박스**(D-0718-0548 — 중앙 모달, 유지).
5. **금지어**: 단정·안심·권유 표현 사전(`src/policy/lexicon.py`)에 걸리는 문구를 **화면 텍스트로 추가 금지**(게이트가 차단한다). 기존 문구는 그대로 두고 스타일만 바꾼다.

## 6. 디자인 레퍼런스 & 실앱 주문 화면 복제 (필수 — 사용자 지정)

- **전체 시각 언어**: `docs/references/image/`의 캡처(26장)를 **정밀히 확인**해 **색상·타이포·여백·버튼/카드/배지 형태를 최대한 유사하게 차용**한다(실앱의 룩앤필 계승).
  - ⚠️ **이 이미지들은 실계좌 정보가 포함된 캡처다. 로컬 참고 전용 — 커밋·푸시 금지.** **디자인(색·레이아웃)만 차용**하고, **실계좌 번호·잔고·보유·개인정보는 데모에 절대 복사하지 않는다.** 데모 표기는 가상 종목 fixture 값만(헌법 §7).
- **실앱 주문 화면 복제**: **S0(진입 주문 화면 재현)**과 **⑧(종착 주문 화면 재현)**은 `docs/references/image/카카오페이증권 어플 캡쳐 (27).jpg`의 **주문 화면 레이아웃을 완전히 재현**한다(구성·색·컴포넌트 배치). 표기 데이터는 **가상 종목 fixture 값만** 사용한다.
- **⑧의 예외(현 기능 유지)**: 모의 체결 이후 다시 진입하는 ⑧ 주문 화면은, **모의 체결 과정에서 투자 판단(투자 일지·선택·다음 재검토)을 기록했다면 그 내용을 함께 표시**한다 — 현재 "내 판단 기록 상기 카드"(s4-ui-spec ⑧) 기능을 **그대로 유지**하며 디자인만 개선한다.
- **재현 명시 라벨 유지**: 두 주문 화면 상단의 "실앱 주문 화면 재현(시연용) — 주문은 실행되지 않아요" 라벨과 "모의 주문 — 실제 거래 아님" 바는 유지한다(재현임을 정직하게 명시 — 규정).

## 7. Claude Design 도구 협업 워크플로

1. **§6의 레퍼런스 이미지(색·컴포넌트)를 기준으로**, Claude Design 도구로 새 화면 디자인(레이아웃·컴포넌트·색·타이포·시각 언어)을 생성/제안받는다.
2. 생성 결과를 **§5 제약에 맞게 번역**해 `app.css`(필요 시 index.html 클래스)에 반영한다:
   - 외부 폰트/CDN/이미지를 쓰면 → **로컬·시스템 폰트·CSS·data URI로 대체**.
   - 게임화·과한 모션·권유 톤 → **제거**.
3. **화면 단위로 점진 반영**: 한 화면 반영 → 브라우저 확인 → 다음 화면. 한 번에 전체를 쏟아붓지 말 것(리스크·롤백 용이성).

## 8. 안전 작업 방식 (원본 보존 — 핵심)

- **반드시 브랜치에서만 작업**: `git checkout -b wt/redesign`. **main은 커밋·푸시된 동작 데모이므로 직접 수정 금지.** 실패 시 브랜치를 버리면 원본이 그대로 복원된다.
  - (원본과 새 디자인을 **동시에 띄워 비교**하려면 worktree: `git worktree add ../redesign wt/redesign` → 원본 9000·재디자인 9200 병행.)
- **작은 커밋**(화면·컴포넌트 단위, 메시지에 무엇을 바꿨는지).
- **매 반영 후 검증(회귀 방지)**:
  1. `pytest -q` → **302 green**(DOM id·구조가 안 깨졌는지).
  2. `run_gate.py` → **통과**(고지·배지·금지어).
  3. 브라우저로 **S0~⑧ 완주** + **콘솔 오류 0** + 안전모드 토글 + DEMO 패널 시나리오 전환.
- **원본 데모가 항상 폴백.** 재디자인이 시연 리스크를 키우면 되돌린다.

## 9. 완료 기준

새 디자인으로 **S0~⑧ + 안전모드 + DEMO 패널이 완주**하고, **pytest 302·안전 게이트·콘솔 오류 0** 통과, **외부 네트워크 요청 0**, **§14·접근성·상시 표기·비가역 확인 체크·실앱 주문 화면(S0·⑧) 재현·⑧ 판단 기록 카드 보존**. 한마디로 — "**보기엔 확 달라졌지만, 기능·안전·정직성은 동일**".
