# 프로젝트 기획 · 판단 여권 (Decision Passport)

> 상세 근거는 리서치 최종 보고(`docs/report/2026-07-17_리서치_최종보고.html`)와 PRD(2026-07-16 대화, 프롬프트 로그 보존)를 따른다. 이 문서는 실행 단계의 요약 기준점이다.

## 문제 정의
초보 투자자(경험·역량 축 정의)는 급락·손실 국면에서 ①왜(근거) ②무엇을 확인해야 하는지(절차) ③선택의 비용·위험·비가역성을 알 수 없어 판단이 정지되거나(손실 방치) 왜곡된다(패닉셀·추격매수). 해법은 결론 대행이 아니라 **출처 있는 판단 재료의 구조화와 선택권의 반환**이다. (근거 3계층: 계좌 실증·VOC 121+118건·출제 인터뷰 — 확신도: 방향 H / 타깃·설계 M / 효과 L)

## 목표와 성공 기준
- 본선 심사(1대1 시연)에서 -8% 급락 시나리오의 8단계 완주 + **안전 게이트 화면 증명**(출처 없는 사실 0·권유 표현 0·기준시각 누락 0, 사전 정의 테스트셋 분모 명시).
- 12:00 초반 1차 제출(심사 순번 확보), 브리프 4항목 + 프롬프트 로그 전량 제출.
- 북극성 지표(설계 목표): "핵심 조건을 이해하고, 계획을 따르거나 근거 있는 변경을 명시한 뒤, 감당 가능한 결정을 완료한 세션 비율" — 보류도 성공, 거래량·수익률 지표 배제.

## 범위
- 포함: 8단계 흐름(계획 회상→관련 사실 브리핑→체크리스트·이해 확인→대칭 시나리오·영향 미리보기→검토 의향·이유 기록→별도 화면 모의 체결→사후 회고→장애 안전모드), 결정론 계산 엔진, 정책 가드, AI 응답 계약, 동결 스냅샷+fixture 폴백, Codex 플러그인 포장.
- 제외(리서치 확정 — CLAUDE.md 14절): 개인화 추천, 방향 결론, 실주문·자동주문, 수익 예측, 게임화, 실시간 의존. 온보딩·모드 선택은 후순위(D-0716-0210-main).
