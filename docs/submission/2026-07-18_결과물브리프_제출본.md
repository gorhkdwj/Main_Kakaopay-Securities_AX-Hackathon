## ① 한 줄 요약

종목 선정을 마친 초보 투자자가 매수·매도·보류의 **마지막 실행 판단** 앞에서, "사라/팔라"를 대신 말하지 않는 **AI 판단 동반자**와 함께 — 출처·기준시각이 붙은 사실, 양면 해석, 대칭 선택지, 비용·세금까지 계산된 시나리오 비교를 보고 — **스스로 설명 가능한 결정**을 내리도록 돕는 '판단 전 브리핑' 시스템.

## ② 문제 정의

- **문제 재정의**: "호가창 앞의 막막함"(How to trade)의 원인을 정보 부족이 아니라 **판단 구조의 부재**로 정의했다. 초보 투자자는 실행 단계, 특히 급락·손실 같은 **고불안 순간에 판단 왜곡이 가장 크다**(자체 리서치 — 출처 402건·주장 123건 교차검증, 확신도 H/M/L 병기).
- **기존 도구의 공백**: 종목 선정 솔루션은 많지만, 마지막 실행 결정은 "지금 사세요/파세요"류 방향 결론이나 커뮤니티 분위기에 맡겨져 있다. 방향 결론은 규제 리스크와 판단 왜곡을 동시에 키운다.
- **우리의 해석**: '매수/매도 지원'을 **방향 제시가 아니라 판단 재료와 선택권의 제공**으로 해석했다. AI는 결론을 말하는 조언자가 아니라 **의견을 나누며 결정을 정교하게 만드는 동반자**여야 한다 — 현장 Q&A에서 확인된 출제 방향("동반자", "사라 팔아라의 의견 제시가 아닌")과 일치한다.
- **대표 장면**: −8% 급락한 보유 종목의 [판매] 버튼을 누른 순간, 결론 대신 판단 재료를 끼워 넣는 '판단 전 브리핑'이 개입한다. 같은 구조가 수익 실현(+15%)·신규 탐색 매수에도 그대로 성립한다(범용 판단 구조).

## ③ 핵심 기능·시연 포인트

**흐름(S0+8단계)**: 실앱 주문 화면 재현(S0)에서 [구매/판매] 클릭 순간 인터셉트 → ① 종목·계획(내 계획 회상) → ② 관련 사실(AI 브리핑) → ③ 체크리스트(+이해 확인 OX) → ④ 시나리오 비교(수량 대칭 비교) → ⑤ 검토 의향(+투자 일지) → ⑥ 모의 주문 → ⑦ 회고 → ⑧ 주문 화면 복귀(실서비스 핸드오프 지점).

**기능 7종 = 시연 포인트**

1. **출처·기준시각 배지** — 모든 사실 카드에 출처 ID·기준시각을 붙이고, 출처가 없거나 실재하지 않는 출처(AI가 지어낸 출처 포함)가 달린 사실 카드는 **렌더 전 차단**한다.
2. **사실 / 해석 / 모름 분리** — AI 응답을 facts(출처 있음)·interpretations(긍정·부정 양면 병기)·unknowns(무엇이 언제 확인되는지)로 구조화해 별도 카드로 표시한다. AI가 상황에 맞는 '다음 질문'까지 제안해 사용자의 검토를 이어가게 한다(동반자 인터랙션의 시작점).
3. **대칭 선택지** — 매수·매도·보류를 기본 강조 없이 대칭 제시. 방향 전환 토글로 같은 급락 화면에서 저가 매수 검토도 동일 구조로 성립한다(양방향).
4. **사전 고지 + 모의 체결** — 수수료·세금·D+2 결제·비가역성 4종 고지를 체결 **전에** 표시하고 비가역 확인 체크 후에만 진행. 별도 화면 모의 체결('모의 주문 — 실제 거래 아님' 상시 표기), 서버가 주문 내용을 결정론 재계산으로 재검증한다.
5. **계산·생성 분리** — 체결·비용·세금·정산·잔여 비중·평단가 등 숫자는 **결정론 계산 엔진**이 만들고, LLM은 근거 붙은 설명만 한다. LLM 응답의 사실(facts) 텍스트 속 숫자는 엔진·데이터의 허용 숫자 집합과 대사해 검증되지 않은 숫자를 차단한다(해석·모름에는 산수 금지 프롬프트 계약과 금지 표현 가드 적용). 수량을 바꿔 넣으면(그대로 유지/일부/전량·N주) 같은 산식으로 즉시 재계산돼 문제 예시의 '한 번에 vs 나누어' 판단을 **수량 대칭 비교**로 지원한다 — 분할 N회 비교는 엔진 산식 추가만으로 확장 가능한 구조다. 손실(−40,989원)·수익(+59,011원)이 같은 산식으로 계산된다(**계산은 방향 중립**).
6. **안전 게이트** — 화면 상단 카운터가 무출처 차단·금지 표현·기준시각 누락을 실시간 누적 표시하고, 제출·시연 전 게이트 스크립트가 안전 테스트셋(위반 27건 전건 차단·정상 21건 오차단 0)과 시나리오 3종 스위프·위험 고지 4종을 일괄 검증한다(당일 실측: 전 항목 통과).
7. **장애 안전모드 / 오프라인 완주** — 네트워크·LLM 실패 시 3계층 폴백(live→cache→static)으로 흐름을 완주하고 현재 원천을 화면 배지로 표시. 외부 CDN·웹폰트·원격 이미지 0(외부 참조 0건 실측).

## ④ 사용 도구·데이터 출처

- **개발 도구**: Claude Code · Codex CLI — 전 과정 AI 협업, 프롬프트 로그 전량 무편집 제출.
