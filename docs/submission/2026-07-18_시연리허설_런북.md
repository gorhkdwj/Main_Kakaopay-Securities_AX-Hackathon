## 1. 브리핑 원천 결정 — live 기본 (현장에서 확인)

② 브리핑의 응답 원천은 화면 상단 "브리핑 생성:" 배지에 항상 표시된다(live / cache / static).

**기본 동선(real_005930)은 live가 기본**이다 — real 시나리오는 LLM 캐시가 없어 live(키 있음, ~20~30초) 또는 static(즉시) 중 하나로 뜬다. **`BRIEFING_MODE=cache`로는 real 브리핑이 static 강등**(지문 캐시 부재)되므로, 라이브 시연은 `.env`의 `ANTHROPIC_API_KEY` 채움 + `BRIEFING_MODE=auto`(또는 `live`)를 확인한다.

| 모드 | 설정 | 특징 | 언제 |
|---|---|---|---|
| **live**(기본) | `.env`에 `ANTHROPIC_API_KEY` 채움, `BRIEFING_MODE=auto` 또는 `live` | 실시간 AI 설명(배지 "live"). 응답 ~20~30초 | 기본 동선(real_005930) — Wi-Fi 안정 + 실AI 강조 |
| **static**(real 폴백) | 키 공백 또는 live 실패 시 자동 | 즉시·네트워크 0. 배지 "static" | real에서 live 불가 시 자동 강등(배지로 정직 표시) |
| **cache**(가상 폴백) | `.env`에 `BRIEFING_MODE=cache` → 서버 재시작 | 네트워크 0·즉시. 배지 "cache" | 백업 동선(loss8 등 가상)에서 무지연 시연 |

- 🖥️ **오프라인 완주는 코드로 증명됨**: 전체 312 테스트 통과(2026-07-18 10:35 실측 — 네트워크 0, TestClient·cache). 키 공백 시 네트워크를 아예 시도하지 않는다.
- ✋ **사전 캐시 워밍(백업 동선 대비, 권장)**: 시연 전 `.\.venv\Scripts\python.exe scripts\briefing\gen_llm_cache.py`로 현재 fixture 지문에 맞는 캐시 확보(키 필요) — 가상 시나리오에서 live 실패 시 cache가 즉시 받는다. fixture를 바꾸면 반드시 재생성(안 하면 지문 불일치로 static 강등). real 시나리오 캐시는 생성되지 않는다(real의 폴백은 static).

2. **재정의(우리의 해석)** — "저희는 이 막막함의 원인을 정보 부족이 아니라 **판단 구조의 부재**로 봤습니다. 특히 급락·손실 같은 고불안 순간에 판단 왜곡이 가장 큽니다 — 이건 저희가 이틀간 출처 402건·주장 123건을 교차검증한 리서치로 확정한 전제입니다."
3. **접근(경계가 곧 설계)** — "그래서 '지금 사세요/파세요'를 말하는 AI가 아니라 — Q&A 키워드 그대로 — **의견을 나누며 결정을 정교하게 만드는 동반자**를 만들었습니다. 원칙은 세 가지입니다: 모든 사실에 출처와 기준시각, 숫자는 AI가 아니라 결정론 엔진이 계산, 방향 결론·권유는 렌더 전에 자동 차단."
4. **서비스 소개(무엇이 있나)** — "실제 주문 화면에서 [판매] 버튼을 누르는 순간 '판단 전 브리핑'이 개입해 — 내 계획 회상 → 사실/해석/모름 브리핑 → 수량 대칭 비교 → 모의 체결까지 완주합니다. 그리고 어느 화면에서든 **동반자 버튼**으로 '나눠 살까, 한 번에 살까?' 같은 질문을 하면 엔진 계산으로 답합니다. 데이터는 야후 파이낸스에서 수집·동결한 **삼성전자 실데이터**이고, 목표주가·투자의견 같은 권유성 필드는 수집 단계에서 차단했습니다."
5. **증거(오늘 실측)** — "말이 아니라 실측으로 증명합니다: 오늘 아침 테스트 361건 전건 통과, 안전 게이트(위반 표현 27건 전건 차단·정상 문장 오차단 0) 통과, 그리고 시연에서 직접 '팔까요?'라고 물어보시면 — **방향 결정을 정중히 거절하는 것까지** 보실 수 있습니다."

### 9-3. 시연 전환 멘트

> "마침 저희가 어제 수집한 스냅샷이 **실제 급락 국면**입니다 — 삼성전자 −8.8%, 이튿날 코스피 −6.4%. 문제문이 말한 바로 그 순간을 실데이터로 보여드리겠습니다." → §4-1 동선 시작.

### 9-4. 발화 시 금지 앵커 (한 줄 자기점검)

- 효과("불안이 줄어든다")를 사실처럼 말하지 않기 — "…하도록 돕는 것이 가설이고, 구조로 증명합니다"까지만.
- "검증된"은 실측 수치(테스트·게이트)에만 붙이기. 미래 수익·정확도 언급 금지.
- "타이밍을 알려주는"이라고 말하지 않기 — "타이밍 판단의 재료를 구조화"로.
