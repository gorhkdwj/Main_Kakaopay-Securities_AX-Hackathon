## 1. 온사이트 모드 결정 — live vs cache (현장에서 택1)

② 브리핑의 응답 원천은 화면 상단 "브리핑 생성:" 배지에 항상 표시된다(live / cache / static).

| 모드 | 설정 | 특징 | 언제 |
|---|---|---|---|
| **live**(기본) | `.env`에 `ANTHROPIC_API_KEY` 채움, `BRIEFING_MODE=auto` 또는 `live` | 실시간 AI 설명(배지 "live"). 응답 ~20~30초 | Wi-Fi 안정 + 실AI 강조하고 싶을 때 |
| **cache**(폴백) | `.env`에 `BRIEFING_MODE=cache` → 서버 재시작 | 네트워크 0·즉시. 배지 "cache" | 현장 Wi-Fi 혼잡/불안, 무지연 시연 원할 때 |

- 🖥️ **오프라인 완주는 코드로 증명됨**: 전체 312 테스트 통과(2026-07-18 10:35 실측 — 네트워크 0, TestClient·cache). 키 공백 시 네트워크를 아예 시도하지 않는다.
- ✋ **사전 캐시 워밍(권장)**: 시연 전 `.\.venv\Scripts\python.exe scripts\briefing\gen_llm_cache.py`로 현재 fixture 지문에 맞는 캐시 확보(키 필요) — 그러면 live 실패 시에도 cache가 즉시 받는다. fixture를 바꾸면 반드시 재생성(안 하면 지문 불일치로 static 강등).
- **원터치 폴백**: 시연 중 live가 느리면 `.env`의 `BRIEFING_MODE=cache`로 바꾸고 서버만 재시작하면 끝.
